# 因为下划线导致latex公式渲染不正确/渲染成斜体

有些hugo的主题会带有latex的渲染插件，
这些插件通常用`$`作为行内公式的标记符。

公式中经常会用到下划线来标记下标，
不巧的是markdown的渲染引擎，
通常也会将下划线作为斜体的标记，
所以会出现被渲染为斜体的情况。

解决的方法比较简单，用反斜杠来转义下划线即可。

```hugo
一些文字$a_b=i$，然后是另一些文字$c_d=j$。 // 这种可能会有问题

一些文字$a\_b=i$，然后是另一些文字$c\_d=j$。 // 这样通常就可以解决了
```
